What happens when you initialize a repository? Why do you need to do it?

	When a repository is initialized, a git file is added to the directory. This needs to be done so that git can recognize it.

How is the staging area different from the working directory and the repository? What value do you think it offers?

	The staging area is different in that it is the work space before committing something. It allows the user to move files around before actually committing.
